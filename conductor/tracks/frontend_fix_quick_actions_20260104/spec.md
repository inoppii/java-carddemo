# 仕様書: フロントエンド機能不全の修正 (Quick Actions)

## 概要
デプロイされたアプリケーションのダッシュボードにある「Quick Actions」セクションのリンクが機能していない問題を調査し、修正する。

## 不具合の詳細
以下の「Quick Actions」リンクをクリックしても、適切な画面に遷移しない、または反応がない。
- View Transactions
- Request Credit Increase
- Report Lost Card

## 調査・修正範囲
1.  **フロントエンド (React):**
    -   `Dashboard.tsx` 等における「Quick Actions」セクションの実装確認。
    -   リンク (`Link` コンポーネントや `onClick` ハンドラ) の設定確認。
    -   遷移先となる各画面コンポーネントの実装有無の確認。
    -   ルーティング設定 (`App.tsx` 等) の確認。
2.  **バックエンド (Spring Boot):**
    -   上記機能に対応する API エンドポイントの実装有無の確認。
    -   必要に応じて API の実装またはスタブ作成。

## ゴール
- 「Quick Actions」内のすべてのリンクが正常に機能し、適切な画面に遷移すること。
- 遷移先の画面が（少なくともプロトタイプとして）表示されること。

## 受け入れ基準
- ダッシュボードの「View Transactions」をクリックすると、取引履歴画面に遷移すること。
- ダッシュボードの「Request Credit Increase」をクリックすると、利用限度額増額申請画面（またはフォーム）に遷移すること。
- ダッシュボードの「Report Lost Card」をクリックすると、紛失届画面（またはフォーム）に遷移すること。
