# プラン: 現行資産の分析とデータスキーマ設計

## フェーズ 1: 現行資産の精密分析
このフェーズでは、COBOL コピーブックと DDL を読み解き、現行のデータ構造を詳細に把握します。

- [ ] **タスク 1:** 顧客・アカウント関連のコピーブック分析 (`CUSTREC.cpy`, `CVACT01Y.cpy` 等)
- [ ] **タスク 2:** トランザクション・カード関連のコピーブック分析 (`CVTRA05Y.cpy`, `CVCRD01Y.cpy` 等)
- [ ] **タスク 3:** DB2 および IMS 関連の定義分析
- [ ] **タスク: Conductor - User Manual Verification 'フェーズ 1: 現行資産の精密分析' (Protocol in workflow.md)**

## フェーズ 2: PostgreSQL スキーマ設計
分析結果に基づき、PostgreSQL 用のモダンなスキーマを設計します。

- [ ] **タスク 1:** 顧客・アカウント・カード情報のテーブル設計
- [ ] **タスク 2:** トランザクション履歴・承認データのテーブル設計
- [ ] **タスク 3:** 参照データ（トランザクションタイプ等）のテーブル設計
- [ ] **タスク 4:** インデックスおよび制約（外部キー等）の定義
- [ ] **タスク: Conductor - User Manual Verification 'フェーズ 2: PostgreSQL スキーマ設計' (Protocol in workflow.md)**

## フェーズ 3: ドキュメント化と検証

- [ ] **タスク 2:** ER図の作成（Mermaid または画像）
- [ ] **タスク 3:** DDL ファイルの生成と基本動作検証
