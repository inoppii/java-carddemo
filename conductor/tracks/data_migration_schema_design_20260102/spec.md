# トラック仕様: 現行資産の分析とデータスキーマ設計

## 目的
現行のメインフレーム環境で使用されているデータ構造（VSAM, DB2, IMS）を深く理解し、それらを Google Cloud SQL (PostgreSQL) 上で効率的かつ拡張性の高いリレーショナルスキーマへと再定義する。これは、アプリケーションのモダナイゼーション（Node.js/Java への移行）における基盤となる。

## 対象となる現行データ資産
- **VSAM:** 顧客マスター、アカウントマスター、カードマスター、トランザクション履歴、ユーザーセキュリティ。
- **DB2:** トランザクションタイプ、カテゴリ。
- **IMS DB:** クレジットカード承認リクエスト、履歴。

## 成果物
1. **データ分析レポート:** 各現行ファイルのフィールド定義、データ型、制約のまとめ。
2. **PostgreSQL スキーマ定義 (DDL):** `CREATE TABLE` 文、インデックス、制約。
3. **データマッピングドキュメント:** 現行フィールドから新テーブル・カラムへの対応表。
4. **ER図:** 移行後のリレーショナルモデルの可視化。

## 制約・考慮事項
- すべてのドキュメントおよびコメントは日本語で作成する。
- 3270 エミュレータ特有のデータ形式（COMP, COMP-3 等）を PostgreSQL の適切な型（NUMERIC, INTEGER 等）へ変換する。
- 移行後の Web UI での利用を考慮し、命名規則をモダンなもの（スネークケース等）に変更する。
